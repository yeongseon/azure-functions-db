# 보안 / 설정

## 1. 원칙

- 비밀은 코드에 넣지 않는다.
- 가능한 경우 managed identity / 환경변수 / secret store 사용
- checkpoint storage 권한은 최소화
- DB 계정은 읽기 전용 우선

## 2. 연결 정보

### Azure Functions 측
- `AzureWebJobsStorage`
- DB connection setting (예: `ORDERS_DB_URL`)

### 권장
- 운영에서는 plain password보다 managed identity 또는 secret reference 사용 검토
- 개발/테스트와 운영 값을 분리

## 3. 최소 권한

### DB 계정
MVP polling에는 보통 다음이면 충분:
- 대상 테이블 SELECT
- 필요 시 metadata/schema read

### Storage 계정
- checkpoint/lease blob read/write
- 필요 시 quarantine container write

## 4. 데이터 최소화

handler에 전달되는 payload는 최소 필요한 컬럼만 투영하는 옵션을 지원해야 한다.
민감 정보 컬럼은 기본적으로 projection에서 제외하는 운영 가이드 권장.

## 5. 로그 민감정보 보호

다음은 로그에 남기지 않는다.
- full connection string
- password/token
- row 전체 민감 payload
- raw secrets

## 6. 멀티테넌트 고려

초기 버전은 app-level trust를 가정한다.
다중 테넌트 DB를 동시에 다룰 경우:
- poller namespace 분리
- checkpoint path 분리
- quarantine path 분리
- per-tenant DB credential 분리

## 7. 공급망

- 의존성 pinning
- lock file 유지
- CI에서 vulnerability scan
- optional drivers는 extra로 분리

예:
- `azure-functions-db[postgres]`
- `azure-functions-db[mysql]`
- `azure-functions-db[mssql]`
