# ADR-005: 통합 패키지 설계 (trigger + binding → azure-functions-db)

## 상태
**승인됨** (2026-04-08)

## 컨텍스트

초기 설계에서는 `azure-functions-dbtrigger`라는 이름으로 DB 변경 감지(trigger)만 제공하는 패키지를 계획했다.
개발 과정에서 다음 요구가 제기되었다:

1. DB 읽기(input binding)와 쓰기(output binding)도 함께 제공하면 사용자 편의가 높아진다
2. trigger + binding을 별도 패키지로 나누면 사용자가 여러 패키지를 설치해야 한다
3. 하나의 패키지(`azure-functions-db`)로 통합하면 설치와 설정이 간단해진다

## 결정

`azure-functions-dbtrigger`를 `azure-functions-db`로 이름을 변경하고,
trigger(변경 감지)와 binding(읽기/쓰기)을 하나의 패키지에 포함한다.

### 내부 구조
- `core/`: 공유 인프라 (EngineProvider, DbConfig, types, errors, serializers)
- `trigger/`: 기존 polling trigger 설계 그대로 유지
- `binding/`: DbReader (input), DbWriter (output)
- 의존 방향: trigger→core, binding→core, trigger↔binding 상호 import 금지

### API 전략
- imperative API (DbReader, DbWriter)가 primary
- decorator sugar (@db.input, @db.output)는 thin wrapper로 나중에 추가
- 패키지는 통합하되, 프로그래밍 모델은 통합하지 않음

## 대안 검토

### 대안 A: trigger-only 유지 (azure-functions-dbtrigger)
- 장점: scope 집중, 네이밍 명확
- 단점: 사용자가 binding 필요 시 별도 구현 또는 별도 패키지 필요
- 기각 이유: 사용자 DX가 떨어짐

### 대안 B: 별도 패키지 (azure-functions-dbtrigger + azure-functions-dbbinding)
- 장점: 관심사 분리 명확
- 단점: 두 패키지 설치 필요, engine/config 중복
- 기각 이유: "여러 개 다운받으면 귀찮다"

### 대안 C: 즉시 통합 API 설계 (decorator 중심)
- 장점: 깔끔한 단일 API surface
- 단점: decorator가 Azure Functions runtime과 충돌 위험, 검증 전 API 고정
- 기각 이유: imperative API 안정화 후 decorator 추가가 안전

## 근거

1. **기술적 시너지**: engine/config/pool 공유가 자연스러움
2. **사용자 편의**: `pip install azure-functions-db` 하나로 모든 DB 작업
3. **scope 관리**: 내부적으로 core/trigger/binding 분리로 복잡도 격리
4. **점진적 확장**: trigger 먼저 구현, binding은 Phase 8+ 추가
5. **Oracle 리뷰**: "하나의 패키지, 하나의 프로그래밍 모델은 아님" 권장 채택

## 시장 조사 결과

- Microsoft Azure Functions SQL Extension: SQL Server 전용, trigger+binding 통합 extension
- 동일 scope의 Python-native 패키지: 현재 존재하지 않음
- 우리 차별점: multi-DB + Python-first + SQLAlchemy + 단일 패키지

## 결과

- 패키지명: `azure-functions-db`
- import: `azure_functions_db`
- 기존 trigger 설계는 그대로 유지
- binding semantics는 별도 문서(22-Binding-Semantics.md)로 정의
- 개발 phasing: trigger (Phase 1-7) → shared core (Phase 8) → binding (Phase 9-11)
