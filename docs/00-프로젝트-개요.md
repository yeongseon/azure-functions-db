# 프로젝트 개요

## 프로젝트명

`azure-functions-db`

## 문제 정의

Azure Functions는 Azure 서비스 중심의 trigger/binding 경험은 좋지만, 범용 DB 전반에 대해
일관된 **trigger + binding 개발 경험**을 제공하지는 않는다. 특히 여러 RDBMS를 함께 쓰는 팀에서는
다음 문제가 반복된다.

- DB별로 polling 코드를 매번 새로 만든다.
- DB row를 읽기 위해 함수마다 연결/세션 코드를 반복 작성한다.
- 함수 결과를 다른 테이블에 쓰기 위해 input/output 패턴을 매번 직접 조립한다.
- checkpoint / dedup / batching / retry 정책이 팀마다 다르다.
- 함수 코드가 비즈니스 로직보다 운영 코드로 오염된다.
- 로컬 개발과 운영 환경의 동작 차이가 크다.
- "트리거처럼 보이는" 추상화가 없어서 팀 내부 사용성도 낮다.

## 해결 방향

`azure-functions-db`는 Azure Functions Python 앱에서 사용할 수 있는 **통합 DB 연동 프레임워크**다.

- 실행 기반: Azure Functions `timer trigger`
- 이벤트 감지: polling, outbox, CDC adapter
- 데이터 읽기: imperative input binding(`DbReader`)
- 데이터 쓰기: imperative output binding(`DbWriter`)
- DB 연결: SQLAlchemy 중심 + 비SQL adapter 확장
- 상태 저장: Blob checkpoint store(MVP), 이후 Table/Cosmos 확장
- 보장 수준: 기본적으로 **at-least-once에 가까운 pseudo trigger**
- 대상 언어: MVP는 Python

## 프로젝트 철학

### 1. 정직한 naming
이 프로젝트는 **native trigger가 아니다**.
문서와 코드에서 `pseudo trigger`, `poll trigger`, `db event trigger`라는 표현을 유지한다.

### 2. 보장 계약을 먼저 정의
지원 DB 수보다 중요한 것은 **실패/재시도/중복/정렬에 대한 contract**다.

### 3. SQLAlchemy는 reach를 넓히는 도구
목표는 ORM이 아니라 **이벤트형 데이터 처리 DX**다.

### 4. 운영 가능한 기본값
로컬에서 편하고, 운영에서 망가지지 않는 구성이 우선이다.

### 5. 작게 시작해서 전략을 늘린다
MVP는 `cursor polling`만 지원하고,
이후 DB별 native capability(change feed / binlog / WAL / outbox)를 추가한다.

### 6. 하나의 패키지로 모든 DB 작업
변경 감지(trigger), 읽기(input binding), 쓰기(output binding)를 분리된 제품이 아니라
**하나의 패키지**로 제공해 Azure Functions 개발자가 DB 작업을 한 가지 방식으로 다루게 한다.

## 비전

Azure Functions 개발자가 다음처럼 생각하게 만드는 것.

> "DB 작업이 필요하면 azure-functions-db 하나만 설치하면 된다."

## MVP 대상

- PostgreSQL
- MySQL / MariaDB
- SQL Server
- Python Azure Functions v2 programming model
- Blob checkpoint store
- Cursor/updated_at polling
- Row dict payload
- DbReader
- DbWriter
- Single-table polling
- Single-function ownership per poller

## 비대상(초기)

- 진짜 custom trigger extension
- Exactly-once 보장
- Cross-table transactional change merge
- DB delete event 완전 자동 감지
- Schema migration 자동 관리
- 강한 전역 ordering
- 멀티테넌트 SaaS control plane
