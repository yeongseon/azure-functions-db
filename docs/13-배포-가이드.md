# 배포 가이드

## 1. 권장 배포 모델

### MVP
- Azure Functions Python app
- Azure Storage account
- 대상 DB
- Application Insights

## 2. 환경 분리

- dev
- staging
- prod

각 환경별로 분리:
- function app
- storage account/container namespace
- checkpoint path
- DB endpoint
- app insights

## 3. host.json 권장

```json
{
  "version": "2.0",
  "extensionBundle": {
    "id": "Microsoft.Azure.Functions.ExtensionBundle",
    "version": "[4.0.0, 5.0.0)"
  },
  "logging": {
    "applicationInsights": {
      "samplingSettings": {
        "isEnabled": true
      }
    }
  }
}
```

## 4. 앱 설정

필수:
- `AzureWebJobsStorage`
- `FUNCTIONS_WORKER_RUNTIME=python`
- `ORDERS_DB_URL`
- `AZFDB_STATE_CONTAINER=db-state`

선택:
- `DBTRIGGER_LOG_LEVEL=INFO`
- `DBTRIGGER_MAX_BATCHES_PER_TICK=1`
- `DBTRIGGER_DEFAULT_BATCH_SIZE=100`

## 6. 업그레이드 원칙

다음 변경은 위험하다.
- source query 변경
- cursor column 변경
- PK 변경
- checkpoint schema 변경

원칙:
- source fingerprint가 달라지면 같은 checkpoint 재사용 금지
- 필요한 경우 migration 또는 reset 계획 수립

## 8. 장애 복구

### storage 일시 장애
- bounded retry
- 장기 장애 시 function fail
- checkpoint 미전진 유지

### DB 장기 장애
- lag 증가 허용
- 복구 후 catch-up
- schedule/batch size 재조정 가능

### 잘못된 배포
- rollback
- checkpoint는 일반적으로 유지
- source contract가 달라졌다면 rollback 후도 fingerprint 확인 필요
